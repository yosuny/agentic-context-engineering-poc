# ACE 성능 검증을 위한 데이터셋 생성 전략

ACE(Agentic Context Engineering)의 핵심 역량인 **지식 누적, 실수를 통한 학습, 전략적 추론**을 효과적으로 증명하기 위한 데이터셋 구축 전략입니다.

---

## 1. 데이터셋 확보 및 도메인 선정

ACE의 지능적 진화를 검증하기 위해 **"공공 데이터의 범용성"**과 **"실무 데이터의 전문성"**을 결합한 하이브리드 전략을 취합니다.

### 🎯 권장 도메인: 제조 안전 및 유지보수 (Manufacturing Safety & Maintenance)
- **이유**: "사고 원인 분석 -> 재발 방지 대책"의 논리 구조가 ACE의 **Reflector(원인 분석)** & **Playbook(예방 전략)** 메커니즘과 완벽하게 일치합니다.
- **데이터 확보**: 한국산업안전보건공단(KOSHA)의 재해 사례집은 공개되어 있으며, 인과 관계가 명확한 텍스트 데이터입니다.

### 📂 주요 데이터 출처 (Open Data Sources)
1. **한국산업안전보건공단 (KOSHA)**:
   - "재해사례집": 사고 개요, 기인물, 발생 형태, 원인, 대책이 구조화된 텍스트.
   - "안전보건 가이드(KOSHA Guide)": 작업별/설비별 안전 수칙 표준 매뉴얼.
2. **AI Hub (aihub.or.kr)**:
   - "산업안전 텍스트 데이터": 위험성 평가 및 사고 예방 관련 문서.

## 2. 데이터셋 계층 구조 (Dataset Hierarchy)

| 단계 | 명칭 | 목적 | 구성 특징 |
| :--- | :--- | :--- | :--- |
| **Phase 1** | **Training Seed (학습셋)** | 플레이북의 초기 지식 및 전략 추출 유도 | 오류 유발 가능성이 높은 복잡한 계산, 예외 조항 포함 |
| **Phase 2** | **Validation Set (검증셋)** | 학습된 전략의 일반화 성능 측정 | Phase 1과 유사한 로직을 요구하지만 수치/상황이 다른 변형 문제 |
| **Phase 3** | **OOD/Stress Set (극한셋)** | 상충하는 규칙이나 극단적 엣지 케이스 대응 | 법 개정 시나리오, 다중 예외 중첩 상황 |

## 2. ACE 특장점별 시나리오 설계 (Scenario Design)

### 2.1 "실수를 통해 배우는가?" (Error-Driven Learning)
*   **시나리오**: 작업자가 설비 점검 중 협착 사고 발생 상황 제시.
*   **검증 포인트**: 초기에는 '단순 부주의'로 판단하다가, 학습 후 **'LOTO(잠금장치) 미이행'**이나 **'방호장치 무효화'** 같은 근본 원인을 지적하고 플레이북에 [Safety-Check] 항목을 추가하는지 확인.

### 2.2 "전략이 정교화되는가?" (Strategy Refinement)
*   **시나리오**: 복합적인 증상(진동 + 소음)이 있는 설비 고장 진단.
*   **검증 포인트**: 단순 부품 교체 제안에서 벗어나, **'베어링 마모 -> 축 정렬 불량'** 등 단계적 진단 절차(Diagnostic Flow)를 플레이북에서 인용하며 추론하는지 측정.

### 2.3 "지식이 압축/유지되는가?" (Context Efficiency)
*   **시나리오**: 수십 페이지의 세법 문서를 요약한 플레이북만으로 복잡한 문제 해결.
*   **검증 포인트**: 전체 문서를 RAG로 참조할 때와 ACE 플레이북(압축된 전략)만 참조할 때의 토큰 효율성 및 답변 정확도 비교.

---

## 3. 데이터 생성 파이프라인 (Generation Pipeline)

1.  **Seed Extraction**: KOSHA 재해사례집(PDF/Text)에서 `기인물`, `작업 내용`, `사고 원인`, `대책`을 구조화된 JSON으로 추출.
2.  **Scenario Augmentation**: 추출된 사례를 기반으로 "김 대리가 컨베이어 벨트를 점검하러 들어갔는데..."와 같은 현장감 있는 **상황 묘사 텍스트(Context)** 생성.
3.  **Gold Standard Logic**: 해당 상황에서 반드시 지켜야 했을 **안전 수칙(Rule)**을 정답 근거로 매핑.

---

## 4. 평가 메트릭 고도화 (Advanced Metrics)

*   **Strategy Hit Rate (SHR)**: 생성된 답변에서 플레이북의 전략 ID가 얼마나 유효하게 인용되었는가.
*   **Reasoning Consistency**: 동일한 로직의 다른 문제(Phase 1 vs Phase 2)에 대해 일관된 전략을 사용하는가.
*   **Delta-Accuracy**: 플레이북 주입 전후의 정답률 차이 ($Accuracy_{Augmented} - Accuracy_{Baseline}$).
